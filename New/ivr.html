Interactive Voice Response (IVR) System Re-Design

Project Description:

All business with customer service lines use a phone tree system of menus and options to help guide the caller to a specialist who can help them resolve their issues. For this project, we were tasked with examining the current IVR system in place, and trying to find ways to help improve it through research, design, and heuristics. In particular, the business wanted us to focus on making the system easier to use since a large portion of the caller demographic are adults over the age of 50.

Heuristic Evaluation

Like any other system, IVR's have core design principles that should be upheld for the best possible experience and usability. Sources such as Google Scholar helped us find articles showing the limits of human patience, ability to memorize and understand options, and mental fatigue. Industry leaders, such as Twilio, Helped break down simple best practices to follow when it comes to designing an IVR.

Heuristic Evaluation Criteria

Message Length

A user should not have to wait more than 30 seconds for a prompt to finish

Menu Options

A user should never have more than 5 options to choose from at once

Live Agent Option

A user should be able to dial out to a live user at any time to help resolve their inquiry

User Inputs Should Transfer

A user should have any inputs passed to the representative so they don't need to enter it again

Allow Barge In

A user should be able to interrupt the system at any point; this is expecially valuable for repeat callers

Offering Multiple Languages

A user should be able to select their preferred language from the beginning of a call

Allow Multiple Data Entry Attempts

A user should be able to ttempt entering data multiple times before erroring out

Follow Standard Key Press Patterns

A user should be able to follow DTMF conventions to navigate the IVR system

Call Transfer Prompts

Prompts related to call transfers should only be heard immediately before the transfer happens

Instructions Before Inputs

A user should be able to hear any instructions before the key to execute them

Leave Marketing Material Out

A user should be able to accomplish their tasks without being bogged down by marketing

Professional Voice and Tone

A user should feel like the system has a professional and friendly tone that reflects the brand

Heuristic Evaluation Findings

Not Following Key Press Standards

Our IVR system does not follow current key press patterns, and worse, often re-assigns what keys do as users navigate through the system

Prompts Are Too Long

Our IVR system has several prompts that exceed 30 seconds which goes against the recommended length of 30 seconds or less

Too Many Menu Choices

Our IVR system often has menus that exceed 5 choices, this makes it harder for users to remember choices and causes them to repeat selections

Marketing Material Clutter

Our IVR system presents marketing material before our callers needs are resolved

Poor Voice and Tone

Our IVR system uses very stiff, unfriendly language, which can make users feel like the company doesn't care about them or their issues

IVR Journey Mapping

To get an accurate picture of our callers experience when using the IVR system, we created a journey map to show the steps involved in a call and the overall mood of our callers during each step. We noticed that most callers grew more frustrated the longer they spent in the IVR system, and only felt better once they were eventually connected to an actual human being after either requesting it or erroring out in the call tree.
